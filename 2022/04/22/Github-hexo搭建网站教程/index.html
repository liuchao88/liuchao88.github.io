Github Pages 静态网页建站

步骤

github配置

新建一个Repositories，Repository name填 你的GitHub用户名.github.io

  • 打开博客网址，提示404“There isn’t a GitHub Pages site here.”，解决方法是进入github仓库的settings，首先把repository visibility设置成public，然后进入Pages，Source中的Branch选择master，Custom domain设置成自己的博客地址，保存后等待TLS验证完毕即可。

Git配置

  • 命令行输入ssh -t git@github.com测试是否是连接至github，弹出提示Warning: Permanently added the RSA host key for IP address '20.205.243.166' to the list of known hosts. PTY allocation request failed on channel 0，
    • 说明没有公钥。
      • 重置用户名和邮箱
        • git config –global user.name “yourname”
        • git config –global user.email“your@email.com“
      • 删除.ssh文件夹下的known_hosts
      • 创建一个公钥，输入ssh-keygen -t rsa -C ‘邮箱地址’，然后设置存放公钥的位置和密码，默认的话直接回车键确认。
      • git查看ssh公钥的方法
        • 1）通过git bash命令窗口
          • 直接输入cat ~/.ssh/id_rsa.pub即可查看
        • 2）直接打开用户目录下的.ssh文件夹，打开里面的id_rsa.pub文件即可查看
      • 将公钥添加到github，点击右上角带黄色的头像–>settings–>（personal settings中的）SSH and GPG keys–>new SSH key–>title自己取名，key中粘贴publickey–>Add SSH key
    • 命令中的-T应该是大写。
    • 在git中输入命令：ssh -T git@github.com然后会跳出一堆话。输入命令：yes
  • 运行‘hexo d’提示remote: Support for password authentication was removed on August 13, 2021. Please use a personal access token instead.意思是自从21年8月13后不再支持用户名密码的方式验证了，需要创建个人访问令牌(personal access token)。
    • 创建令牌：点击Settings→点击左侧的Developer settings→点击Personal access tokens→点击Generate new token，选择scope全勾上即可。
    • 应用令牌

Hexo

  • 运行hexo时报错“The “mode” argument must be integer”，原因是hexo版本和Node版本不匹配，Node版本过高。解决方案是将hexo更新到新版本，或者将Node退回到旧版本。

  • hexo升级时报错use_date_for_updated，解决方案是找到hexo的配置文件：_config.yml 更改其中的：use_date_for_updated 为 updated_option

  • 运行hexo d报错 Failed to connect to github.com port 443 after 21101 ms，是因为hosts中没有添加github的ip地址，参考
